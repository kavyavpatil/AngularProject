Announcement Details

Please check the below FAQ's to know more about the User Role feature. What are User Roles? User roles is a feature that allows the Fleet Administrator to create custom roles based on a set of permissions and UI configurations.Once created, they can assign one of these custom roles to a given user. The user assigned to a specific role will inherit all the permissions and UI configurations that have been given to the defined role.

This feature allows one to manage user permissions in an organized or structured way. It is immensely helpful when one has to manage a large number of users in the system/portal. How does it affect me as a portal user? With the introduction of user roles in production deployment, the end user on the fleet portal may see some changes to the UI or in terms of the functionality that he/she has seen in the past. The user may no longer see some pages for which they no longer has permission to.

If the user does not see the page through which permissions to other users were given (applies to admin users in the fleet who had such access to earlier), he/she will have to contact the TSP customer support for more information. Does it require any change from my side? Yes, there are some actions required on your side if you are a user of the portal based on what role you belong to. Please check below what action is required from your end based on your role:

If you are a Fleet Administrator - Please share the list of users who act as Fleet Administrators in your fleet to the TSP/service provider.

If you are a Fleet Manager or any other role - You will be assigned the role of a Fleet Manager. This will give you all the permissions except for User Management. For more information, contact your Administrator. What if I don't want to use User Roles? From the next release onwards User roles feature will be the default way to manage users for all portals. The legacy way of managing permissions to each user in the portal will be deprecated. By default, we will follow the below rule to assign roles to users:

Administrator Role - Any user who has access to user management

Fleet Manager Role - Any user who doesn't have access to user management Whom to contact if I face any issues with the new feature? Please contact your Administrator/Service Provider, if you are facing any issues with your login post-user role feature is released in production.
